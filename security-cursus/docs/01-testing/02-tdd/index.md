# Les 3: Test Driven Development

In Les 1 schreven we eerst de klasse, dan pas de tests. In deze les draaien we dat volledig om. We schrijven eerst de test, ook al bestaat de klasse nog niet, en pas daarna de implementatie. We bouwen verder op ShopWave en voegen een `CouponService` toe via TDD.

## Leerdoelen

Na deze les kan je:

- uitleggen wat Test Driven Development is en waarom het over ontwerp gaat, niet over testen
- de drie wetten van Uncle Bob toepassen tijdens het coderen
- de Red-Green-Refactor cyclus stap voor stap doorlopen
- vooraf een testlijst opstellen voor een nieuwe klasse
- een klasse volledig via TDD bouwen in C# met xUnit
- TDD combineren met Dependency Injection en Moq uit Les 1

## Wat heb je nodig?

**Installeer dit voor je begint:**
- Het ShopWave-project en testproject uit Les 1
- NuGet-pakketten: `xunit`, `Moq`, `FluentAssertions`
